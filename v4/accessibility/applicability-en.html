WCAG 2.0 success criteria applicability

Work in progress

This page is a work in progress and requires review. Please file an issue or submit a pull request if information or coding is missing, incorrect or out of sync with the main repository (wet-boew/wet-boew).

Looking for WET v3.1?

As of September 23, 2014, version 3.1 of the Web Experience Toolkit is no longer supported. The source code and documentation have been moved to the wet-boew-legacy repository.

The following is a list of the Web Content Accessibility Guidelines (WCAG) 2.0 success criteria and the most common applicable types of web page content for each. Note that this list is not exhaustive and is intended as a guide only.

1. Perceivable

1.1.1 Non-text content

  • Images
  • Forms
  • Video
  • Audio

1.2.1 Audio-only and Video-only (Prerecorded)

  • Video (no audio)
  • Audio (no video)
  • Animated Images

1.2.2 Captions (Prerecorded)

  • Video (with Audio)

1.2.3 Audio Description or Media Alternative (Prerecorded)

  • Video (with Audio)

1.2.4 Captions (Live)

  • Video (with Audio)

1.2.5 Audio Description (Prerecorded)

  • Video (with Audio)

1.3.1 Info and Relationships

  • Tables
  • Forms
  • Headings
  • Quotations
  • Lists

1.3.2 Meaningful Sequence

  • All Content

1.3.3 Sensory Characteristics

  • All Content

1.4.1 Use of Color

  • Images using Colour
  • Forms using Colour Labels
  • Tables using Colour

1.4.2 Audio Control

  • Audio

1.4.3 Contrast (Minimum)

  • All Content using Colour

1.4.4 Resize text

  • All Text Content

1.4.5 Images of Text

  • Images
  • Forms using Images

2. Operable

2.1.1 Keyboard

  • Forms via Keyboard
  • Links via Keyboard
  • Plug-ins

2.1.2 No Keyboard Trap

  • Forms via Keyboard
  • Links via Keyboard
  • Plug-ins

2.2.1 Timing Adjustable

  • Forms
  • Web Applications

2.2.2 Pause, Stop, Hide

  • Animated Images
  • Video
  • Audio
  • Any content that changes or moves automatically.

2.3.1 Three Flashes or Below Threshold

  • Animated Images
  • Video

2.4.1 Bypass Blocks

  • Links

2.4.2 Page Titled

  • Page Title

2.4.3 Focus Order

  • Forms via Keyboard
  • Links via Keyboard

2.4.4 Link Purpose (In Context)

  • Links

2.4.5 Multiple ways

  • Links

2.4.6 Headings and Labels

  • Headings
  • Forms

2.4.7 Focus Visible

  • Forms via Keyboard
  • Links via Keyboard

3. Understandable

3.1.1 Language of Page

  • HTML element

3.1.2 Language of Parts

  • Quotations
  • Any content where a change of natural language occurs

3.2.1 On Focus

  • Forms via Keyboard
  • Links via Keyboard

3.2.2 On Input

  • Forms via Keyboard

3.2.3 Consistent Navigation

  • Links

3.2.4 Consistent Identification

  • Forms
  • Links

3.3.1 Error Identification

  • Forms

3.3.2 Labels or Instructions

  • Forms

3.3.3 Error Suggestion

  • Forms

3.3.4 Error Prevention (Legal, Financial, Data)

  • Forms

4. Robust

4.1.1 Parsing

  • All Content

4.1.2 Name, Role, Value

  • Forms
